# 慢病用药小管家

## 一、项目简介

“慢病用药小管家”是一款面向慢性病患者的前端网页应用 Demo，旨在帮助用户在完成线上复诊和购药后，建立连续的个人用药管理流程。

慢性病患者在长期用药过程中，常见问题包括忘记按时服药、重复购药、药品库存不足、复诊前缺少用药记录整理等。本项目围绕“问诊—购药—用药—续方”的服务闭环，设计了用药计划、服药打卡、库存提醒、复诊提醒和复诊小抄等功能。

本项目为课程/比赛场景下的产品原型 Demo，重点展示产品结构、核心流程和交互设计。

---

## 二、项目背景

慢性病患者通常需要长期、规律地服药，并定期复诊开药。但在线上复诊和互联网购药流程中，患者往往缺少一个连续的管理工具，导致以下问题：

1. 线上复诊后，患者容易忘记医生建议和具体用药安排；
2. 药品购买后，缺少库存管理，容易出现重复购买或断药；
3. 每日服药缺少提醒和打卡机制，容易漏服；
4. 复诊前缺少用药记录整理，医生难以及时了解患者近期用药情况。

因此，本项目设计了一个轻量化的“慢病用药小管家”网页应用，帮助用户完成从用药计划到复诊准备的连续管理。

---

## 三、核心功能

### 1. 今日用药计划

用户可以查看当天需要服用的药品，包括药品名称、服用时间、剂量和服药状态。

### 2. 服药打卡

用户点击“已服药”后，系统会记录服药状态，并自动更新今日计划。

### 3. 药品库存管理

用户每完成一次服药打卡，系统会模拟扣减药品库存。当库存不足时，页面会显示提醒，提示用户及时续方或购药。

### 4. 药品信息管理

用户可以查看、添加或编辑药品信息，包括药品名称、用法用量、服药频率、库存数量等。

### 5. 复诊提醒

系统根据用户设置的复诊日期，展示复诊倒计时，提醒用户提前准备复诊材料。

### 6. 复诊小抄生成

系统根据用户的服药记录、库存情况和近期用药状态，生成一份复诊前参考内容，帮助用户与医生沟通。

## 四、技术栈

本项目采用纯前端技术实现：

- React
- Vite
- TypeScript
- localStorage
- CSS / Tailwind CSS

---

## 五、项目特点

1. **纯前端实现**  
   当前版本不接入后端数据库，不需要真实登录系统，便于快速部署和演示。

2. **使用 mock 数据模拟真实场景**  
   项目内置慢病患者、药品、服药计划和复诊信息等模拟数据，用于展示完整业务流程。

3. **使用 localStorage 保存操作记录**  
   用户在浏览器中的服药打卡、药品编辑等操作可以在本地保存，刷新页面后仍可保留部分数据。

4. **聚焦产品闭环设计**  
   项目重点验证“线上问诊后建立用药计划—每日服药打卡—库存消耗—续方提醒—复诊准备”的完整流程。

## 六、运行方式

### 1. 安装依赖

```bash
npm install
```
### 2. 本地运行
npm run dev

运行后，在浏览器中打开终端显示的本地地址，例如：

http://localhost:3000/
### 3. 构建项目
npm run build

构建完成后会生成 dist 文件夹，可用于部署到 Vercel、Netlify 等静态托管平台。
